# kaizen — 自己改善ループの棚卸し

## なぜこのスキルがあるか

失敗や成功をその場で台帳（docs/LESSONS.md）に書く運用は、書きっぱなしでは資産にならない。定期的に台帳を見直し、繰り返しパターンを「恒久ルール」や「スキル」に昇格させて初めて、実行のたびに賢くなるループが完成する。これは記事でいう Skills 自動生成の役割を、Human-in-the-Loop（ユーザー承認つき）で行うもの。

## 手順

### 1. 情報収集
- docs/LESSONS.md / docs/STATE.md / docs/DECISIONS.md を読む
- `git log --oneline -20` で直近の作業を確認する

### 2. 記録漏れの補完
- 直近のコミット・この会話の中に、台帳に載っていない失敗（ビルドエラー、修正指摘、手戻り）や成功パターンがあれば LESSONS.md に追記する
- STATE.md を実態に合わせて整理する（完了したものは完了ログへ、完了ログは直近5件まで）

### 3. 昇格提案（3回ルール）
- LESSONS.md で ×3 以上になったパターンを抽出し、A/B のラベル付き選択肢で提案する:
  - 絶対ルール化: DECISIONS.md「現在の前提・絶対ルール」へ追記
  - スキル化: .claude/skills/<name>/SKILL.md を新設、または既存スキル（ship 等）へ手順追記
- ユーザーが選んだものだけ反映する。見送った場合もその判断を LESSONS.md に一言残す（同じ検討の繰り返しを防ぐ）
- 勝手に恒久ルール化・自動化しない（Human-in-the-Loop 原則）

### 4. 報告
- 「台帳に追記したもの / 昇格を提案するもの / 変化なし」を3〜5行の日本語で報告する
- docs/ や .claude/skills/ に変更があればコミットする（push は ship スキル経由でのみ行う）
