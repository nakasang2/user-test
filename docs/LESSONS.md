# LESSONS.md — 失敗・成功パターン台帳

> Claude向け運用ルール: ビルド/デプロイの失敗、ユーザーからの修正指摘、同じ質問の繰り返しが起きたら**対応した直後に**1エントリ追記する。同種の作業を始める前に該当カテゴリを読む。同種パターンの発生回数（×N）が3回に達したら、DECISIONS.md の絶対ルール化またはスキル化を /kaizen で提案する。

フォーマット: `日付 | 事象 → 原因 → 回避策`（1〜3行。再発したら新規追加せず既存エントリの ×N を加算）

## 失敗パターン

### ビルド・デプロイ
- 2026-06 | Vercel ビルド内の `prisma db push` が P1017（DB接続失敗）で落ちる ×3 → pooler 経由の接続はビルド時に不安定 → db push は directUrl（直結）経由で実行し `--accept-data-loss` を明示（コミット 5824e2f → e52c851 → f50946c の3回で確立。絶対ルールに昇格済み）
- 2026-05 | push 後に本番ビルドエラーが初めて発覚し、エラーログを貼ってもらうループが発生 ×多数 → push 前のローカル検証の省略 → `npx tsc --noEmit` + `npx next build` を必ず push 前に実行（ship スキルに昇格済み＝この台帳の昇格ルールの成功例）

### 品質・レビュー
- 2026-06 | 多エージェントレビューでトークン漏えい・タスク進行バグが確定指摘（8d545e7） → 実装した本人（同じ視点）のセルフチェックだけでは見落とす → push 前に別視点レビュー（/code-review）を ship スキルの手順に組み込み

### UX/UI・録画
- 2026-07-16 | Canvas合成録画で PiP（顔映像）を固定比率（4:3）の矩形に描画し、16:9カメラが縦に潰れて保存されていた（widget/page.tsx） → drawImage の描画先矩形をソース実比率と無関係に固定していた → PiP高さは必ず `videoHeight/videoWidth` から毎フレーム算出する（InterviewRoom.tsx:860 が正しい実装）。カメラ/映像を扱うUIは「実映像の比率」で検証する
- 2026-07-16 | カメラ/マイク権限を拒否すると、再試行UIが案内オーバーレイの下に隠れ・開始ボタンが永久disabledでデッドエンド（InterviewRoom） → エラー要素とオーバーレイが同一親の兄弟でz-index未指定、かつ `<video>` をエラー時にアンマウントして再取得時に再アタッチ不可 → 権限系の復帰UIは最前面（z-30）に、メディア要素は常時マウントしてエラーはオーバーレイで重ねる
- 2026-07-16 | 公開ページ（`/`）が認証なしで全組織の件数・被験者名を表示し、かつ DB 不通時に 500 クラッシュ → SSRで無条件に prisma を叩いていた → 公開ルートでは組織データを扱わない（静的マーケティングページ化）。ページのデータ露出範囲は「未認証で見えてよいか」で判断する
- 2026-07-16 | フォーム送信/クリップボード/削除系で res.ok 未確認・catch欠如の「サイレント失敗」が多発（モーダル・招待・再分析・コピー） → 成功を仮定して UI を更新していた → fetch は必ず res.ok 判定 + try/catch、破壊的操作（失効・削除）はサーバー確定後にのみ UI 反映する
